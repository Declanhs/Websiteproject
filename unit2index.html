1.5 Describing copyright and other constraints affecting websites

You do not need to apply for copyright, copyright is automatic after creation if you sign your work or not. There is a system called Copyscape that will compare webpages and return any with matching text. Copyright means you need the right to copy someone else’s work and if you do not get this you can and probably will be sued a fairly decent amount of money. Another thing to watch out for is GDPR this limits how long you can keep sometimes data and what you are gong to use it for. If you break GDPR you can get in a lot of trouble and be fined

1.6 Describing access issues that need taking into account - describe issues that people with disabilities may face when using web pages and how we can overcome them

There are a fair few things that can be done to accommodate people with disabilities such as being visually impaired. For example, allowing enlarged text, be mindful of the colour, shortcuts to help navigation and allow desktop users to view site in mobile site. You could also enable a siri like text reader what can read text if you hover over it with the mouse, this would allow the person to know exactly what was going on.

1.7 Describe which are the appropriate filetypes for websites

The appropriate file for a website are; jpeg for imagines, pdf which is just for a text document you can’t edit, mp4 is video and sound, and PNG is the filetype that is usually used for transparent image. A Jpeg is the type of file you would use to attach your images to a website
